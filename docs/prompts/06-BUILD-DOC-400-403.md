## Critical Rules

1. **Do NOT create database migrations.** Tables, functions, enums, indexes, and RLS policies all exist.
2. **Do NOT modify `.env.local`, `package.json` deps, or CI workflows.**
3. **Public forms require NO authentication.** The UUID token IS the authorization.
4. **Use SECURITY DEFINER functions for all token-based access** — `get_validation_by_token()`, `submit_validation_response()`, `get_evaluation_by_token()`, `submit_evaluation_response()` via `supabase.rpc()`.
5. **Use `spendCoins()` from `src/lib/coins/actions.ts`** for coin deductions — do not write raw coin transaction SQL.
6. **Compute `overall_rating` client-side** for evaluations: `(sum of 5 ratings) / 5`, rounded to 2 decimal places.
7. **Email sending is deferred.** Console.log the token URL for MVP. Add a `// TODO: Send transactional email` comment.
8. **Mobile responsive required.** External validators will open links on phones.
9. **Professional design.** These public forms are the first impression of Grey Sky for external people.
10. **Self-review gate (5 checks) is mandatory before reporting done.**
